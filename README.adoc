= luainstaller
:toc: macro
:toclevels: 2
:sectanchors:
:sectlinks:

`luainstaller` turns a Lua entry script and its `require` dependencies into a
native executable bundle. It is intended for deployment to a known target
environment: the target must match the bundle's operating-system family,
architecture, system ABI, and Lua ABI.

The project provides two command names:

* `luai` is the short, flag-based interface for scripts and quick use.
* `luainstaller` is the descriptive interface for interactive use and logs.

Both commands call the same Lua API and produce the same bundles.

The supported interpreter range is Lua `>= 5.4` and `< 5.5`; in practice every
installer, discovery process, header set, and linked runtime must identify as
Lua 5.4. LuaJIT and other Lua major/minor ABIs are rejected. A linked-runtime
probe runs before publication; Windows cross-builds therefore require Wine in
addition to MinGW.

[#quick-start]
== Quick start

Install from LuaRocks:

[source,sh]
----
luarocks install luainstaller
luai -v
----

Or install directly from a checkout:

[source,sh]
----
sh tools/install-source.sh --prefix "$HOME/.local"
export PATH="$HOME/.local/bin:$PATH"
luainstaller version
----

Start with the small runtime sample. Build the directory form first because it
keeps the manifest, generated C source, and copied native libraries visible.

[source,sh]
----
lua test/runtime_bundle/main.lua direct
luai -a test/runtime_bundle/main.lua --max-deps 120
luai -t test/runtime_bundle/main.lua --max-deps 120
luai -b --dir test/runtime_bundle/main.lua \
  -o build/runtime-demo --max-deps 120
build/runtime-demo/runtime-demo Ada
----

After the directory bundle works, produce a self-extracting executable:

[source,sh]
----
luai -b --file test/runtime_bundle/main.lua \
  -o build/runtime-demo-onefile --max-deps 120
build/runtime-demo-onefile Ada
----

Expected output begins with `hello Ada`.

[#compatibility]
== Compatibility contract

A bundle does not contain an emulator or an ABI translation layer. Build and
target environments must agree on all four of these boundaries:

* operating-system family;
* CPU architecture;
* system and compiler ABI;
* Lua major/minor ABI.

The generated launcher enforces `LUA_VERSION_NUM == 504`, and Linux additionally
requires `pkg-config --modversion lua` to report 5.4. This is a same-environment
packager, not a hermetic sandbox: embedded modules win, but an undeclared module
may fall through to the target host's normal Lua/native searchers. Always run a
release candidate once with `LUA_PATH` and `LUA_CPATH` cleared.

Pure-Lua modules are embedded in the launcher. Lua C modules are copied into the
bundle, but their dependent system libraries are not discovered recursively.
Check them with `ldd`, `otool -L`, or a Windows dependency viewer before release.

Current target profiles are:

[cols="1,2,3",options="header"]
|===
|Target |Build profile |Release test environment

|Linux x86_64
|Shared Lua runtime
|Native Linux x86_64 host

|Linux ARM64
|Shared Lua runtime
|NVIDIA DGX Spark (`aarch64`)

|macOS ARM64
|Static Lua launcher
|Apple Silicon Mac Studio

|Windows x86_64
|MinGW launcher with Lua DLL
|Wine and Windows 10 x64 VM
|===

See xref:docs/PLATFORMS-NATIVE-LIMITS.adoc[Platforms and native modules] for
toolchain requirements and unsupported combinations.

[#command-summary]
== Command summary

[cols="1,2,2",options="header"]
|===
|Task |Short command |Descriptive command

|Help
|`luai -h`
|`luainstaller help`

|Version
|`luai -v`
|`luainstaller version`

|Analyze
|`luai -a app/main.lua`
|`luainstaller analyze app/main.lua`

|Trace resolution
|`luai -t app/main.lua`
|`luainstaller trace app/main.lua`

|Build
|`luai -b app/main.lua`
|`luainstaller build app/main.lua`

|Read logs
|—
|`luainstaller logs`
|===

The syntaxes are deliberately separate. `luai build app.lua` and
`luainstaller -b app.lua` are rejected.

[#development]
== Development baseline

From the repository root:

[source,sh]
----
find src test -type f -name '*.lua' -print0 | xargs -0 -n1 luac -p
for script in tools/*.sh; do sh -n "$script"; done
lua test/cli_split_smoke.lua
lua test/contract_docs.lua
lua test/production_edges.lua
lua test/smoke_all.lua
luarocks lint luainstaller-1.0.0-1.rockspec
----

`test/contract_docs.lua` checks promises made by the public documentation.
`test/production_edges.lua` covers fail-closed filesystem behavior, source
mutation, SHA-256 ownership, concurrency, ABI rejection, and reproducibility.
`test/smoke_all.lua` exercises dependency discovery, generated launchers,
native modules, installed commands, onedir, onefile, and output safety.
