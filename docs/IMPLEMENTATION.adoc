== Pipeline

[source,text]
----
entry.lua
   |
   v
require discovery
   |  static analyzer, manual includes, or runtime require trace
   v
dependency plan
   |  Lua source files + native Lua C module files + trace records
   v
manifest
   |  normalized paths, hashes, Lua/platform info, compatibility notes
   v
Lua bootstrap + C launcher
   |  embedded Lua payload + native module filesystem paths
   v
onedir bundle
   |
   +--> optional onefile extractor wrapper
----

== Module Map

[cols="1,2,3",options="header"]
|===
|Module |Role |Contract

|`src/init.lua`
|Public API facade.
|Normalizes options, returns structured result tables, dispatches
analysis, trace, compatibility, and bundle operations.

|`src/cli.lua`
|Command-line parser and renderer.
|Implements split `luai` and `luainstaller` command personalities.

|`src/analyzer.lua`
|Static dependency analyzer and resolver.
|Returns script/native dependency lists and trace records, or structured
errors.

|`src/require_engine.lua`
|Discovery strategy coordinator.
|Chooses static/manual/runtime plan, applies manual include/exclude
rules.

|`src/manifest.lua`
|Manifest builder.
|Creates schema version 1 manifest with normalized paths and hashes.

|`src/runtime.lua`
|Pure Lua runtime helper.
|Installs bundled Lua searcher and runs payload with controlled `arg`.

|`src/cgen.lua`
|Lua bootstrap generator.
|Builds payload table and bootstrap source for launcher embedding.

|`src/launcher.lua`
|C launcher generator.
|Embeds bootstrap bytes and emits shared-Lua launcher C source.

|`src/bundler.lua`
|Onedir bundler.
|Validates profile/output, compiles launcher, copies runtime/native
files, writes manifest.

|`src/onefile.lua`
|Self-extracting wrapper builder.
|Stages onedir, embeds files in extractor C, compiles onefile output.

|`src/platform.lua`
|Host and target profile helper.
|Reports target OS, executable suffix, native extensions, loader rpath,
and Lua prefix.

|`src/compat.lua`
|Compatibility diagnostics.
|Returns summary, notes, warnings, host/target, Lua ABI, mode, and
native count.

|`src/logger.lua`
|Persistent log storage.
|Stores up to 1000 Lua-serialized log entries under `~/.luainstaller`.
|===

== Dependency Discovery

The default `static` engine uses a Lua lexer focused on require
extraction. It is not a full Lua parser; it is scoped to require
discovery.

Recognized examples:

[source,lua]
----
require("model")
require 'storage'
require([[reports]])
pcall(require, "optional.module")
----

Rejected examples:

[source,lua]
----
local name = "model"
require(name)
require("plugin." .. kind)
----

The resolver ignores known Lua built-ins, expands names through
`package.path` and `package.cpath`, records candidate paths, classifies
selected files as Lua source or native module, and recursively scans Lua
source dependencies.

== Discovery Engines

[cols="1,3",options="header"]
|===
|Engine |Behavior

|`static`
|Lexes source files, extracts literal require calls, resolves through
Lua and native search paths, and recurses.

|`manual`
|Starts with no automatic dependencies. Adds only explicit `include`
files, then applies excludes. Manual includes are treated as Lua source
dependencies.

|`runtime`
|Writes a temporary tracer, wraps `require`, runs the entry once, records
actual module names, and resolves those records. Coverage is limited to
the executed build-time path.
|===

Trace records include `requiring_file`, `source_line`, `requested`,
`optional`, `candidates`, `selected_path`, `selected_type`,
`classification`, and `reason`.

== Manifest Contract

`.luai/manifest.lua` is executable Lua data:

[source,lua]
----
-- generated by luainstaller
return {
  version = 1,
  hash_algorithm = "fnv1a32",
  entry = { ... },
  output = { ... },
  lua = { ... },
  platform = { ... },
  launcher = { ... },
  modules = {
    lua = { ... },
    native = { ... },
    external = {},
  },
  manual = { ... },
  trace = { ... },
  compatibility = { ... },
}
----

Manifest generation rejects duplicate destinations with
`DuplicateModuleError`. Package init files preserve relative paths, for
example `.luai/lua/a/init.lua` and `.luai/lua/b/init.lua`.

== Runtime Generation

[source,text]
----
cgen.buildPayload
  -> read entry and Lua dependency source
  -> map module names to source records
  -> emit Lua bootstrap

launcher.generateSource
  -> convert bootstrap string to C byte array
  -> append launcher C template
----

The generated C source is retained at `.luai/build/launcher.c`.

The launcher creates a Lua state, opens standard libraries, builds
`arg`, installs traceback handling, loads the embedded bootstrap, and
calls it. The bootstrap configures native module search paths, installs
a bundled Lua source searcher at searcher position 2, strips BOM and
shebang lines from bundled source, and runs the entry chunk.

== Public Error Strategy

Public API functions return `{ ok = false, error = ... }` for normal
user errors. Internal functions may throw structured tables; `init.lua`
converts them into public result tables.

== Logging

Logs are serialized as Lua source under:

[source,text]
----
~/.luainstaller/logs.lua
----

Entries contain timestamp, level, source, action, message, and details.
The logger keeps the newest 1000 entries and supports filtering by
level, source, and action.
