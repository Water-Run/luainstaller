== Mode Summary

[cols="1,2,2,2",options="header"]
|===
|Mode |CLI |Output |Tradeoff

|Onedir
|`luai -b --dir entry.lua -o dist/app`
|A directory containing the launcher and `.luai/` support files.
|Best for inspection and native-loader debugging.

|Onefile
|`luai -b --file entry.lua -o dist/app`
|A single native extractor executable.
|Best for distribution after the inner onedir layout has been validated.
|===

== Onedir Layout

[source,text]
----
dist/app/
    app
    .luai/
        manifest.lua
        native/
            native modules and runtime files
        build/
            launcher.c
----

Build steps:

[arabic]
. validate target profile and output path;
. generate `.luai/build/launcher.c`;
. copy native modules into `.luai/native/`;
. compile the launcher;
. copy or record Lua runtime files;
. write `.luai/manifest.lua`.

== Launcher Runtime

[source,text]
----
OS loader
  -> generated launcher executable
     -> Lua C API
        -> embedded bootstrap
           -> bundled Lua searcher
           -> native cpath patterns
           -> entry chunk
----

Bundled Lua modules are loaded by a searcher inserted at position 2:

[source,text]
----
1. preload searcher
2. bundled luainstaller source searcher
3. normal Lua source searcher
4. normal native C searcher
----

Native path patterns are prepended relative to the launcher:

[source,text]
----
<launcher-dir>/.luai/native/?.so
<launcher-dir>/.luai/native/?/init.so
<launcher-dir>/.luai/native/?.dylib
<launcher-dir>/.luai/native/?/init.dylib
<launcher-dir>/.luai/native/?.dll
<launcher-dir>/.luai/native/?/init.dll
----

== Onefile Build Flow

Onefile output is an extractor around a staged onedir bundle:

[arabic]
. validate onefile output path;
. create a temporary onedir staging directory;
. build the normal onedir bundle into staging;
. enumerate staged files;
. read file bytes and compute FNV-1a hashes;
. generate `extractor.c` with embedded file arrays;
. compile the outer executable;
. remove temporary staging/build directories.

== Onefile Runtime

[cols="1,2",options="header"]
|===
|Platform |Cache root

|POSIX
|`${TMPDIR:-/tmp}/luainstaller-onefile/<payload-id>/`

|Windows
|`%TEMP%\luainstaller-onefile\<payload-id>\`, then `%TMP%`, then current
directory fallback.
|===

For each embedded file, the extractor removes unsafe symlink/reparse
targets, reuses existing files whose size and hash match, creates parent
directories, writes changed files, applies executable mode on POSIX, and
then starts the inner launcher.

[IMPORTANT]
====
Onefile is not an in-memory native module loader. Extraction is required
because `.so`, `.dylib`, `.dll`, and Lua runtime files must exist on the
filesystem before platform loaders can use them.
====

== Output Safety

Onedir output is rejected when:

* output is `/`, `.`, empty, current working directory, or symlink;
* output exists and is not a directory;
* output is a non-empty directory without a generated manifest;
* output has a generated manifest but unexpected top-level files.

Onefile output is rejected when:

* output is `/`, `.`, empty, current working directory, or symlink;
* output exists and is a directory.

A directory is recognized as generated only when `.luai/manifest.lua`
starts with `-- generated by luainstaller` and top-level entries are
limited to expected generated names.

== Debugging Order

[arabic]
. Build `--dir`.
. Run the generated executable directly.
. Inspect `.luai/manifest.lua`.
. Inspect `.luai/native/`.
. Use `ldd`, `otool -L`, or DLL tools on native files.
. Build `--file` only after onedir works.
. If onefile fails, inspect the extraction cache and inner launcher.
