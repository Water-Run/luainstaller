Diagnose the earliest failing step. Confirm the directory bundle before the
single-file form.

CLI examples use `luai`. Logs use `luainstaller logs`.

toc::[]

[#troubleshooting-first]
== First five steps

[source,bash]
----
lua app/main.lua --help
luai -a app/main.lua --max-deps 120
luai -t app/main.lua --max-deps 120 --verbose
luai -b --dir app/main.lua -o build/app --max-deps 120
EMPTY_PATH=$(mktemp -d /tmp/luainstaller-empty-path-XXXXXX)
env -i PATH="$EMPTY_PATH" build/app/app
rmdir "$EMPTY_PATH"
----

Record the first failure.

[#troubleshooting-cli]
== Command rejected immediately

Messages such as `unknown luai command` / `unknown option`:

* `luai` uses `-a` / `-t` / `-b` — not `luai build`
* options that take values must be followed by a value
* runtime arguments for the entry go after `--`

[source,bash]
----
luai -b app/main.lua -- --profile staging
----

[#troubleshooting-errors]
== Error types

[cols="2,3",options="header"]
|===
|Type |Meaning

|`ScriptNotFoundError`
|Entry or include cannot be opened

|`LuaSyntaxError`
|Source fails the text compiler

|`SourceChangedError`
|Source bytes changed after discovery

|`DynamicRequireError`
|Static mode cannot prove a computed `require`

|`ModuleNotFoundError`
|No candidate under search templates

|`DependencyLimitExceededError`
|Exceeded `--max-deps`

|`DuplicateModuleError`
|Module name or bundle path collision

|`InvalidOptionsError`
|Illegal options

|`InvalidOutputError`
|Output cannot be safely replaced (only v2 auto-rebuild)

|`UnsupportedPlatformError`
|`--target-os` disagrees with the host

|`UnsupportedLuaVersionError`
|Not a supported official 5.x (including LuaJIT)

|`ToolchainError`
|Compiler / prefix / `pkg-config` / ABI mismatch; bad explicit prefix is not
auto-replaced

|`CompilationFailedError`
|Compiler ran and failed

|`LuaRuntimeNotFoundError`
|Linux cannot find shared liblua

|`DiscoveryError`
|Runtime discovery failed or incomplete

|`FilesystemError`
|Write, rename, cache, or similar failure
|===

[#troubleshooting-static]
== Analysis disagrees with runtime

[source,bash]
----
lua -e 'print(package.path); print(package.cpath)'
----

Analyze with the same environment. Do not run the source under one Lua and
`luai` under another.

Resolution is entry-rooted before `package.path`. Bare `lua app/main.lua` does
not add the entry directory by itself.

For static mode, watch concatenated names, require aliases, path setup before
the entry, and non-standard optional loads. `luai -t --verbose` shows
candidates.

[#troubleshooting-runtime]
== Runtime discovery

[source,bash]
----
lua app/main.lua profile_a input.json
luai -a app/main.lua -d runtime -- profile_a input.json
----

Only this run's branch is observed. Missing modules swallowed by `pcall` are
not dependencies. Do not use a side-effecting entry as a build-time tracer.

[#troubleshooting-compile]
== Compile / link

**Linux**

[source,bash]
----
pkg-config --cflags --libs lua
cc --version
lua -v
----

**macOS**: check `include/lua.h`, `lib/liblua.a`, and architecture under
`--lua-prefix`.

**Windows**: in an MSVC + Windows SDK environment, check `cl.exe` and
`lua*.dll` under the prefix.

[#troubleshooting-native]
== Native module fails after start

Confirm `.luai/native`, manifest trace, architecture and system libraries;
clear Lua environment variables; test on the real target OS.

[source,bash]
----
file build/app/.luai/native/*
ldd build/app/.luai/native/*.so
EMPTY_PATH=$(mktemp -d /tmp/luainstaller-empty-path-XXXXXX)
env -i PATH="$EMPTY_PATH" build/app/app
rmdir "$EMPTY_PATH"
----

If it only works without a clean environment, the process is using host
modules—that does not prove packaging.

[#troubleshooting-output]
== Rebuild refused

[source,bash]
----
find build/app -print | sort
sed -n '1,160p' build/app/.luai/generated-output.txt
----

The first line must be `luainstaller-generated-output-v2`, and the tree must
match recorded SHA-256 digests. v1 requires manual delete or a new `-o`.

Lock directory: `build/.luai-lock-<hash>`. After confirming no builder is
running, remove the lock indicated by `owner.*`.

[#troubleshooting-onefile]
== Single-file extraction fails

Confirm the directory bundle first. Check `TMPDIR` and
`luainstaller-onefile-<uid>` ownership/mode. Cache hits compare full bytes, not
size alone.

[source,bash]
----
rm -rf "${TMPDIR:-/tmp}/luainstaller-onefile-$(id -u)"
----

[#troubleshooting-logs]
== When reporting issues

[source,bash]
----
luai -v
luainstaller logs --level error --limit 20
lua -v
----

Include: full command and exit status, error type / compiler output, OS and
architecture, Lua version, discovery mode, redacted trace, and whether a clean
environment still fails. Strip secrets and private paths.
