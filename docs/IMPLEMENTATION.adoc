[#implementation-flow]
== Request path

[source,text]
----
CLI / Lua API
  -> validate options
  -> discover dependencies
  -> compatibility diagnostics
  -> manifest
  -> directory or single-file bundle
----

Orchestration is `src/init.lua`. The CLI only parses and renders.

[#implementation-api]
== API

Exports: `analyze`, `trace`, `compatibility`, `bundle`, `getLogs`,
`clearLogs`. The structured result contract applies to `analyze`, `trace`,
`compatibility`, and `bundle` only. Success: `ok = true`. Failure: `ok = false`
and `error`. `getLogs` returns a list of log records; `clearLogs` returns a
boolean. Unknown options for the structured operations are rejected before
discovery.

[#implementation-discovery]
== Discovery

**static**: lex literal require / optional pcall; entry directory first;
computed names → `DynamicRequireError`.

**runtime**: private directory + tracer; official Lua 5.1+; reject LuaJIT,
native loader paths, and modules already cached before the trace. When the
runtime returns **loader data** naming a readable `.lua`, that path is
recorded. A single complete completion marker is required.

[#implementation-manifest]
== Manifest and ABI

Pure-Lua paths stay relative to the entry; outside the tree goes under
`external/`. Manifest v2 uses SHA-256; mismatch → `SourceChangedError`.
Launcher compile-time guards and `_VERSION` probes use the build
interpreter's major/minor ABI and run on the native host.

Ownership markers use `luainstaller-generated-output-v2` and recursive
SHA-256 inventories.

[#implementation-onedir]
== Directory transaction

Private `0700` staging → backup previous output → rename → revalidate →
quarantine delete. The marker records the final path. Overwrite decisions use
the full v2 tree hashes.

[#implementation-onefile]
== Single-file

Only safe relative paths. POSIX: `openat` / `O_NOFOLLOW` and UID cache.
Windows: no reparse follow; handles held until the child exits. Payload ID:
sorted path + mode + size + content SHA-256.

[#implementation-errors]
== Errors

Prefer actionable types: `ScriptNotFoundError`, `LuaSyntaxError`,
`SourceChangedError`, `DynamicRequireError`, `ModuleNotFoundError`,
`DuplicateModuleError`, `InvalidOutputError`, `ToolchainError`,
`CompilationFailedError`, `FilesystemError`, `DiscoveryError`,
`UnsupportedPlatformError`, `UnsupportedLuaVersionError`, and related.

Quote every path in shell commands; reject dangerous compiler-flag
metacharacters.

[#implementation-change-checklist]
== Before changing code

. Add a test that fails on the old behavior
. Four local suites plus syntax checks
. Review generated C when launcher/extractor changes
. Run affected multi-version / multi-platform gates
. Update docs when public behavior changes
. Failed rebuild must not damage prior output
. Clean environment must not rely on host modules
