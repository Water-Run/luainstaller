= luainstaller
:toc: macro
:toclevels: 3
:sectanchors:
:sectlinks:
:icons: font

`luainstaller` packages Lua projects into same-environment executable
bundles. It provides dependency analysis, trace diagnostics, manifest
generation, C launcher generation, onedir bundles, and self-extracting
onefile bundles for the verified Linux, macOS, and Windows sample
matrix.

toc::[]

== Status

[IMPORTANT]
====
The runtime promise is *same OS family, same architecture, same ABI, and
same Lua ABI*. Verified bundles do not require a separate `lua` command
on the target machine, but native Lua C modules and their external
shared libraries must still be compatible.
====

[cols="1,3",options="header"]
|===
|Area |Current state

|Use case
|Package Lua applications into same-environment executables for local
distribution, lab deployment, and controlled runtime environments.

|Command surfaces
|`luai` is compact and script-oriented. `luainstaller` is full-word and
human-oriented. Both call the same Lua API.

|Dependency discovery
|Static require analysis, manual include/exclude mode, and runtime
require tracing are implemented.

|Bundle modes
|`--dir` / `--onedir` and `--file` / `--onefile` are implemented for the
verified sample matrix.

|Target profiles
|Linux same-host shared-Lua profile, macOS static-Lua-prefix profile,
and Windows-from-Linux MinGW profile.

|Main limitation
|Universal cross-platform output and automatic external native-library
closure are not claimed.
|===

== Quick Start

Install with LuaRocks:

[source,sh]
----
luarocks install luainstaller
luai -h
----

Install from a source checkout:

[source,sh]
----
sh tools/install-source.sh --prefix "$HOME/.local"
export PATH="$HOME/.local/bin:$PATH"
luai -v
luainstaller version
----

Try analysis, tracing, and both bundle modes:

[source,sh]
----
luai -a test/student_management_system/main.lua --max-deps 250
luai -t test/student_management_system/main.lua --max-deps 250
luai -b --dir test/runtime_bundle/main.lua -o build/runtime-demo --max-deps 120
build/runtime-demo/runtime-demo world
luai -b --file test/runtime_bundle/main.lua -o build/runtime-onefile --max-deps 120
build/runtime-onefile world
----

== Project Note

`luainstaller` was previously a Python library that could bundle only
pure Lua scripts. The current Lua implementation focuses on dependency
analysis, native-aware bundle layout, and same-environment runtime
contracts.

Project repository:
https://github.com/Water-Run/luainstaller[Water-Run/luainstaller].
