= Web SQL sample

This sample is the service-oriented packaging target. It combines an embedded
browser UI, JSON API, Pegasus/LuaSocket server, native JSON module, optional
Firebird adapter, startup polling, and child-process shutdown.

The default mock driver is sufficient for tests; no Firebird server is needed.

== Dependencies

Basic server:

[source,sh]
----
luarocks install pegasus
luarocks install lua-cjson
----

Optional integrations:

[source,sh]
----
luarocks install luasql-firebird   # real Firebird connection
luarocks install luaossl           # server crypto capability
----

`server.lua` is the entry and route table. `config.lua` reads environment
configuration. `db.lua` implements mock and Firebird adapters. `history.lua`
tracks queries, `security.lua` reports available security capabilities, and
`web.lua` contains the browser page.

== Run in mock mode

[source,sh]
----
FIREBIRD_WEB_SQL_PORT=9090 \
FIREBIRD_WEB_SQL_TOKEN='replace-this-token' \
lua test/firebird_web_sql/server.lua
----

Open `http://127.0.0.1:9090/` and enter the same token. API requests send it in
`X-Auth-Token`.

The Web interface includes a schema browser, SQL editor, result table, history,
favorites, and CSV export. The mock adapter exposes `EMPLOYEE` and `DEPARTMENT`
data for repeatable tests.

== Real Firebird connection

[source,sh]
----
FIREBIRD_WEB_SQL_DRIVER='luasql-firebird' \
FIREBIRD_HOST='127.0.0.1' \
FIREBIRD_DATABASE='/srv/firebird/example.fdb' \
FIREBIRD_USER='app_user' \
FIREBIRD_PASSWORD='from-a-secret-store' \
FIREBIRD_WEB_SQL_TOKEN='replace-this-token' \
lua test/firebird_web_sql/server.lua
----

Do not put production credentials in shell history, source files, logs, or a
bundle. Supply them through the deployment environment or a secret manager.

== Security boundary

The token is application authentication, not transport security. Browser
WebCrypto or optional application-layer encryption does not protect against an
active attacker who can replace the served JavaScript. Use loopback-only access
or HTTPS through a trusted reverse proxy for sensitive databases.

The sample is not an Internet-facing database administration product. Review
query authorization, token storage, TLS termination, audit retention, and
database privileges before adapting it beyond the packaging test.

== API used by the fixture

* `GET /` — embedded browser page;
* `GET /api/status` — server, connection, and security capabilities;
* `POST /api/connect` — select driver/connection values;
* `POST /api/query` and `POST /api/execute` — run SQL;
* `GET /api/tables` and `GET /api/table/<name>` — schema/sample data;
* `GET /api/history` and `POST /api/history/clear` — query history;
* `POST /api/export/csv` — query-to-CSV response.

== Test and package

[source,sh]
----
lua test/firebird_web_sql/smoke_test.lua
luai -t test/firebird_web_sql/server.lua --max-deps 300
luai -b --dir test/firebird_web_sql/server.lua \
  -o build/firebird-web-sql --max-deps 300
----

Probe a packaged server:

[source,sh]
----
FIREBIRD_WEB_SQL_PORT=19090 \
FIREBIRD_WEB_SQL_TOKEN=testtoken \
build/firebird-web-sql/firebird-web-sql &
pid=$!
curl -fsS http://127.0.0.1:19090/api/status \
  -H 'X-Auth-Token: testtoken'
kill "$pid"
----

Expected core discovery includes project modules, Pegasus, LuaSocket, and
`cjson`. Firebird and crypto modules are optional until their corresponding
configuration paths are used.
