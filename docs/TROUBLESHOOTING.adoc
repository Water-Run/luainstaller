== First Diagnostic Pass

[source,sh]
----
luainstaller trace path/to/main.lua --max-deps 250
----

If a bundle exists, inspect its manifest:

[source,sh]
----
lua -e 'local m = dofile("dist/app/.luai/manifest.lua"); print(m.output.mode); print(#m.modules.lua); print(#m.modules.native)'
----

For native loader issues:

[cols="1,2",options="header"]
|===
|Platform |Command

|Linux
|`ldd dist/app/app`

|macOS
|`otool -L dist/app/app`

|Windows
|Check executable directory and `.luai/native/` for required `.dll`
files; use a DLL dependency inspector when needed.
|===

== Common Errors

[cols="1,2,3",options="header"]
|===
|Error or symptom |Likely cause |Action

|`ScriptNotFoundError`
|Entry path or included path does not exist.
|Check working directory, path spelling, and `--include`.

|`DynamicRequireError`
|Static analyzer found a non-literal module name.
|Use `-r runtime` with representative `--` arguments, or use manual
includes for Lua source files.

|`ModuleNotFoundError`
|Required module cannot be resolved through Lua search paths.
|Run the source program with the same `LUA_PATH` / `LUA_CPATH`; install
the dependency or fix working directory.

|`DependencyLimitExceededError`
|Dependency graph exceeded `--max-deps`.
|Raise `--max-deps` only after confirming trace output is expected.

|`ToolchainError: pkg-config cannot find lua`
|Linux bundle build cannot locate Lua compiler/linker flags.
|Install Lua development metadata and confirm
`pkg-config --cflags --libs lua`.

|`ToolchainError: Lua prefix is required`
|macOS or Windows target selected without `--lua-prefix` or
`LUAI_LUA_PREFIX`.
|Point to a target Lua prefix, not an unrelated host prefix.

|`CompilationFailedError`
|Generated launcher or extractor failed to compile.
|Read returned `command` and `output`; verify compiler, headers,
libraries, and profile.

|Generated bundle starts but native module fails to load
|Native module file exists but one external shared library is missing or
incompatible.
|Inspect `.luai/native/`, platform loader output, and the module's own
external dependencies.

|Onefile reuses old extracted files
|Existing extracted file size and hash match the embedded payload.
|Expected behavior. Delete the content-addressed cache only when
debugging extraction.

|`InvalidOutputError`
|Output path is unsafe or contains user data.
|Choose an empty output path or inspect unexpected files before retrying.
|===

== Dependency Discovery Problems

Static discovery works best for literal names:

[source,lua]
----
local model = require("model")
----

It cannot prove arbitrary dynamic names:

[source,lua]
----
local name = os.getenv("APP_MODULE")
local mod = require(name)
----

Use runtime discovery:

[source,sh]
----
luai -a app/main.lua -r runtime -- demo-module
luai -b app/main.lua -r runtime -o build/app -- demo-module
----

Or manual Lua source dependencies:

[source,sh]
----
luai -b app/main.lua --no-depscan \
  --include app/demo-module.lua \
  -o build/app
----

== Toolchain Problems

Linux:

[source,sh]
----
cc --version
pkg-config --cflags --libs lua
lua -e 'print(_VERSION)'
----

macOS prefix must contain:

[source,text]
----
<prefix>/include/lua.h
<prefix>/lib/liblua.a
----

Windows from Linux:

[source,sh]
----
x86_64-w64-mingw32-gcc --version
test -f "$LUAI_LUA_PREFIX/include/lua.h"
test -f "$LUAI_LUA_PREFIX/bin/lua54.dll" || test -f "$LUAI_LUA_PREFIX/lua54.dll"
----

== Native Module Loader Problems

Native Lua C modules must be real files. They cannot be loaded from
embedded Lua strings.

[source,sh]
----
find dist/app/.luai/native -maxdepth 3 -type f | sort
ldd dist/app/.luai/native/socket/core.so
----

For dotted modules such as `socket.core`, expect nested copies:

[source,text]
----
.luai/native/core.so
.luai/native/socket/core.so
----

== Onefile Cache Problems

Cache roots:

* POSIX: `${TMPDIR:-/tmp}/luainstaller-onefile/<payload-id>/`
* Windows: `%TEMP%\luainstaller-onefile\<payload-id>\` or fallback roots

The extractor creates parent directories, removes unsafe symlink or
reparse-point targets, reuses matching files by size/hash, writes
changed files, applies executable mode on POSIX, and runs the inner
launcher.

== Logs

[source,sh]
----
luainstaller logs --limit 20 --level error
----

== Issue Data

Include:

* `luai -v` and `luainstaller version`;
* `lua -e 'print(_VERSION)'`;
* operating system and architecture;
* exact command;
* full error output;
* `luainstaller trace <entry.lua> --max-deps ...` output;
* relevant `.luai/manifest.lua` sections;
* platform loader output for native failures.
