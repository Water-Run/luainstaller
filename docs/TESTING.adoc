[#testing-local]
== Local

From the repository root, with the official interpreter under test:

[source,bash]
----
LUA=${LUA:-lua}
"$LUA" test/lua_abi.lua
"$LUA" test/version_contract.lua
"$LUA" test/cli_split_smoke.lua
"$LUA" test/contract_docs.lua
"$LUA" test/production_edges.lua
"$LUA" test/build_interruption.lua
"$LUA" test/onefile_lifecycle.lua
"$LUA" test/distribution_licenses.lua
"$LUA" test/reproducible_artifacts.lua
"$LUA" test/smoke_all.lua
luarocks lint luainstaller-1.0.0-1.rockspec
----

[cols="2,3",options="header"]
|===
|Suite |Covers

|`test/lua_abi.lua`
|Exact official-Lua ABI and builtin-module capability table

|`test/version_contract.lua`
|Exact pinned patch/source identity, compatibility helpers, and source parsing

|`test/cli_split_smoke.lua`
|`luai` vs `luainstaller` grammar split

|`test/contract_docs.lua`
|Public documentation contracts

|`test/production_edges.lua`
|Boundary and safety-related behavior

|`test/build_interruption.lua`
|SIGINT finalization plus SIGKILL dead-owner recovery and retry

|`test/onefile_lifecycle.lua`
|POSIX process identity/signals and Windows child containment

|`test/distribution_licenses.lua`
|Notices, corresponding source, no-Lua extraction, tamper repair, and relinking

|`test/reproducible_artifacts.lua`
|Byte-identical cross-root artifacts, path privacy, and executable `arg[0]`

|`test/smoke_all.lua`
|Samples, discovery, generated C, directory/single-file, and more
|===

Everything under `src/` must parse on every supported major/minor. Shared test
helpers live under `test/support/`.

[#testing-install]
== Install

[source,bash]
----
luarocks install luainstaller
luai -v
----

Lua 5.5 needs LuaRocks `>= 3.13.0`. Release practice includes isolated tree
install, invoking commands outside the checkout, and running artifacts with
`LUA_PATH` / `LUA_CPATH` cleared. Missing `luarocks` is a failure.

Archives are verified with SHA-256 before extract. The POSIX runner accepts
either `sha256sum` or the macOS-provided `shasum`; a missing verifier is a hard
failure. Built Lua and LuaRocks prefixes carry exact cache-schema markers that
bind the pinned patch archive, build recipe, host class, and compiler. A cache
with a missing or different marker is rebuilt rather than accepted by ABI name
alone.

The POSIX matrix also builds the real sample dependency set separately for
each Lua ABI. It installs only the following SHA-256-pinned source inputs with
LuaRocks dependency resolution disabled, then load-tests every Lua and native
module from that ABI's isolated tree:

The native-dependency cache identity binds both this source list and its exact
installation order. The runner extracts lsqlite3 and the pinned SQLite
amalgamation into distinct roots, rejects any selected source outside its
expected root, and records each archive name, SHA-256, internal C source member,
cache identity, and cache hit/rebuild result in every evidence log. A cold
compile additionally records both exact extraction paths; a cache hit never
pretends that its already-removed build directory still exists.
Direct Linux matrix runs default to the strict `full` edge-coverage mode and
set `LUAI_REQUIRE_FULL_EDGE_COVERAGE=1`, so missing sanitizer, alternate
compiler, LuaJIT, PE cross-inspection, `/dev/full`, or permission probes fail
the local release gate instead of appearing as a pass. Physical-host runners
set `LUAI_MATRIX_EDGE_COVERAGE_MODE=available`: they still run the complete
production edge suite and every probe supported by that host, while each
evidence log explicitly lists unavailable strict prerequisites. A physical
host that has every prerequisite automatically runs the same strict gate.

Each source-built Lua prefix publishes its own `lua.pc`, and the runner pins
`PKG_CONFIG_PATH` to that file. Native fixtures therefore compile against the
headers and libraries for the ABI currently under test, never against a host
`pkg-config lua` entry. The C probe calls a distinct API introduced or specific
to each of Lua 5.1 through 5.5 and is then loaded by the produced executable.
Linux requires `pkg-config` for the full metadata-fallback gate. A clean macOS
host does not: the runner validates the generated `lua.pc` directly and builds
through the explicit source-built Lua prefix, so Homebrew is not a prerequisite.

[source,bash]
----
HOST_LABEL=<label> sh tools/test-lua-versions.sh
----

Windows:

[source,powershell]
----
powershell.exe -NoLogo -NoProfile -NonInteractive `
  -ExecutionPolicy Bypass -File .\tools\test-lua-versions.ps1 `
  -HostLabel <label>
----

A release requires all five versions. Every POSIX version runs the complete
local, native, interruption, onefile, license/relinking, reproducibility,
production-edge, and smoke gates. Every Windows version runs all applicable
native Windows gates, including lifecycle, license/relinking, and
reproducibility. Each per-version evidence log must contain both `lua ABI
capabilities ok` and the final `PASS` record; `version_contract.lua` also
matches the interpreter's exact patch banner to the pinned source digest.

[#testing-orchestration]
== Orchestration (maintainers)

[source,bash]
----
sh tools/remote-test-linux.sh
MAC_HOST=<host alias from local lab config> sh tools/remote-test-macos.sh
----

On Windows run `tools/test-lua-versions.ps1` on the target machine. Transfers
use `git archive`; temporary paths are constrained under patterns such as
`/tmp/luainstaller-*`. Default SSH targets in scripts are lab-local and are not
part of the user-facing contract.

[#testing-real-world-benchmarks]
== Real-world benchmark matrix (manual)

For realistic shipping scenarios beyond the contract suites, run:

[source,bash]
----
sh tools/benchmark-real-world.sh
----

The matrix covers:

* baseline (hello world)
* pure Lua multi-module
* dynamic require (`--require-engine runtime`)
* native C modules (`lua-cjson`, `LuaSocket`)
* luafilesystem (`luafilesystem`)
* `neovim` `runtime/lua` scale stress
* synthetic large trees (`100`, `1000`, `10000` modules)

Useful options:

* `--skip-external` to skip GitHub network projects (`tinykeep`, `neovim`)
* `--skip-native` to skip native module install/cases
* `--fake-scales "100 1000 10000"` to adjust pressure sets
* `--keep-workspace` to preserve fixtures and logs
* `--neovim-timeout 30` to cap each neovim command

Default behavior cleans `/tmp/luainstaller-*` workspace after completion.
Summary and logs are written to:

[source,bash]
----
/tmp/luainstaller-benchmark-*/results/summary.txt
----

[#testing-release]
== Release checklist

. rockspec lint and isolated install
. Production sources parse on all five pinned versions
. Every required local suite above without required skips
. Five-version matrix on each verified environment class
. Directory and single-file runs with `PATH` set to a private empty directory,
  without system `lua`, and without `LUA_PATH` / `LUA_CPATH`
. Repeated single-file build and extract checks
. Byte-identical cross-checkout artifacts and bundled-license relinking checks
. Native modules and system libraries inspected on target
. Documentation does not prescribe cross-build or a device allowlist
. `git diff --check` and clean-checkout review

The candidate is the tip of `main`. Save the command output used for the
decision.
