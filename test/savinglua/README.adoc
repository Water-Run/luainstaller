= savinglua sample

`savinglua` is a small SQLite-backed JSON record store. It exercises a nested
Lua module, two native LuaRocks dependencies, persistent files, and command-line
argument forwarding.

== Dependencies

[source,sh]
----
luarocks install lua-cjson
luarocks install lsqlite3
lua -e 'require("cjson"); require("lsqlite3"); print("dependencies ok")'
----

Both native modules must match the Lua major/minor ABI used for the build and
the target's native ABI.

== Use

[source,sh]
----
db=/tmp/savinglua.sqlite3
lua test/savinglua/main.lua --db "$db" \
  put users:ada '{"name":"Ada Lovelace","score":98}'
lua test/savinglua/main.lua --db "$db" get users:ada
lua test/savinglua/main.lua --db "$db" scan users:
lua test/savinglua/main.lua --db "$db" delete users:ada
----

`main.lua` is the entry. `src/savinglua/store.lua` initializes SQLite and owns
put/get/scan/delete operations. `bench/` and `csrc/` are reserved for future
sample-specific experiments and are not current runtime dependencies.

== Package and verify

[source,sh]
----
lua test/savinglua/smoke_test.lua
luai -t test/savinglua/main.lua --max-deps 250
luai -b --dir test/savinglua/main.lua \
  -o build/savinglua --max-deps 250

EMPTY_PATH=$(mktemp -d /tmp/luainstaller-empty-path-XXXXXX)
env -i PATH="$EMPTY_PATH" build/savinglua/savinglua \
  --db /tmp/bundled-savinglua.sqlite3 \
  put check '{"ok":true}'
rmdir "$EMPTY_PATH"
----

Expected discovery is `savinglua.store`, `cjson`, and `lsqlite3`. The database
path is runtime state and is not packaged.
