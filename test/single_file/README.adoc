= Single-file samples

These thirty independent programs are the cheapest packaging inputs in the
repository: no project modules and no third-party native dependencies. Use them
to separate entry/launcher failures from dependency discovery failures.

[source,sh]
----
lua test/single_file/01_hello_luainstaller.lua
luai -b --dir test/single_file/01_hello_luainstaller.lua \
  -o build/single-hello
build/single-hello/single-hello
----

The numbered files progress from console output, arithmetic, branches, loops,
strings, and file I/O through sorting, recursion, stacks, queues, trees, graphs,
matrix operations, dynamic programming, LRU caching, and topological sorting.
Each file must remain runnable and understandable on its own.

Check the full set:

[source,sh]
----
for file in test/single_file/*.lua; do
  luac -p "$file"
  lua "$file" >/tmp/luainstaller-single.out
done
----

When adding a basic regression fixture, prefer a new self-contained file here
over adding a multi-module sample. Use the larger sample directories only when
the behavior requires module paths, native libraries, persistence, or services.
